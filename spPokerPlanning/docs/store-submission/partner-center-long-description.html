Sprint Align is a SharePoint Framework (SPFx) solution for SharePoint Online that helps Scrum teams, product owners, and delivery leads run planning poker and other agile estimation sessions directly inside Microsoft 365 — without exporting backlog data to a separate SaaS tool.

Deploy the solution once to your SharePoint Online tenant app catalog, add the web part to a modern SharePoint page or publish it as a full-page app, and use the built-in setup wizard to automatically provision the required SharePoint lists. Sprint Align provides join-by-code sessions, private voting, facilitator reveal, round statistics (average, median, range, and outliers), custom card decks, session history, CSV export, and site-level branding and governance — while keeping all session data securely within your Microsoft 365 tenant.

Ideal for
Scrum teams, agile coaches, product owners, delivery leads, and any organization that estimates story points, confidence, or priorities during sprint planning, backlog grooming, or refinement — and wants estimation to stay inside SharePoint and Microsoft Teams.

Key Features

  • Planning poker with Fibonacci and custom card decks
  • Six vote types in one app: planning poker, confidence (1–5), fist-of-five, roman votes, dot voting, and quick surveys
  • Join-by-code sessions — participants enter a six-character code or open a shared link; identity comes from Microsoft 365 automatically
  • Private voting until the facilitator reveals the round together
  • Live participation status so facilitators know when everyone has voted
  • Round results with average, median, range, and outlier highlighting
  • Facilitator controls: reveal, re-vote, skip, advance items, and end session
  • Custom site-level card decks via the deck editor
  • Session history with velocity-by-sprint-tag chart
  • CSV export of session results for reporting
  • Full-page immersive layout with optional SharePoint chrome hiding for workshop-style sessions
  • Light and dark mode with custom brand colors and editable home-page hero text
  • Demo workshop mode to explore the experience before provisioning (no data saved to SharePoint)
  • Unified Settings hub: setup, governance, branding, home page, layout, subscription, and advanced options

Microsoft Teams Integration
After deployment, administrators can use Sync to Teams from the SharePoint App Catalog to make the solution available in Microsoft Teams as a personal app or channel tab. Participants join the same estimation session from Teams or SharePoint — backed by the same SharePoint lists on your site.

Optional Integrations (feature flags)

Site administrators can enable optional capabilities in Settings → Advanced. Core estimation does not require data to leave SharePoint:

  • Microsoft Graph profile photos (optional): Shows participant profile photos in the team roster. Requires Microsoft Graph User.ReadBasic.All, approved once by a tenant administrator in SharePoint Admin Center → API access.
  • Microsoft Graph presence (optional): Shows Teams presence indicators for participants. Requires Microsoft Graph Presence.Read.All, approved once in SharePoint Admin Center → API access.
  • Azure DevOps backlog import (optional): Import work items and write agreed story points back to Azure DevOps. Requires Azure DevOps user_impersonation, approved once in SharePoint Admin Center → API access. Disabled by default.

Security & Data Residency

  • 100% tenant-resident: Sessions, votes, decks, and settings are stored in standard SharePoint lists inside your Microsoft 365 tenant
  • No external estimation backend: Core planning poker and voting do not require data to leave SharePoint
  • Microsoft 365 security inherited: Uses SharePoint permissions, Microsoft Entra ID authentication, conditional access, and DLP policies
  • You own the data: Uninstalling leaves your SharePoint lists and session history intact

Component Information (SPFx component types in this package — policy 1170.1)

  • Client-side web part — "Sprint Align" — for modern SharePoint pages and (via Sync to Teams) Microsoft Teams channel tabs and personal apps
  • Full-page app — the same "Sprint Align" web part published as a SharePoint full-page application (SharePointFullPage host) for a dedicated immersive estimation workspace

Prerequisites

  • Microsoft 365 with SharePoint Online (SharePoint Server / on-premises is not supported)
  • Tenant administrator permissions to deploy SPFx solutions to the app catalog
  • Site owner permission for first-time list provisioning
  • Modern SharePoint pages (SharePoint Framework 1.21)
  • Microsoft Teams (optional) for Teams tab and personal app integration
  • Azure DevOps (optional) for backlog import and story-point writeback

Limitations & Out of Scope

  • SharePoint Online only — not supported on SharePoint Server or on-premises farms
  • One license per site collection — data and configuration are scoped to the licensed site collection
  • Setup wizard required — a site owner must run first-time list provisioning before sessions can be saved to SharePoint
  • No native mobile app — runs in supported desktop and mobile browsers and Microsoft Teams
  • Azure DevOps import is optional — core estimation works without Azure DevOps or any external integration
  • Subscription required after trial — a 14-day free trial is included; a yearly subscription is required to continue after the trial

Recommended Browsers

  • Microsoft Edge
  • Google Chrome

Licensing
